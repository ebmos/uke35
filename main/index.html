DYNAMIC HOST CONFIGURATION PROTOCOL

Dynamic Host Configuration Protocol (DHCP) er en nettverksprotokoll som brukes til å automatisere prosessen med å konfigurere enheter på IP-nettverk. Dette gjør det mulig for dem å bruke nettverkstjenester som DNS, NTP og hvilken som helst kommunikasjonsprotokoll basert på UDP eller TCP. En DHCP-server tildeler dynamisk en IP-adresse og andre nettverkskonfigurasjonsparametere til hver enhet i et nettverk, slik at de kan kommunisere med andre IP-nettverk. DHCP er en forbedring av en eldre protokoll som kalles BOOTP.

DOMAIN NAME SYSTEM

Domain Name System (DNS) er internettets telefonkatalog. Mennesker får tilgang til informasjon på nettet gjennom domenenavn, som for eksempel nytimes.com eller espn.com. Nettlesere samhandler gjennom internettprotokoll (IP)-adresser. DNS oversetter domenenavn til IP-adresser slik at nettlesere kan laste internettressurser.

Hver enhet som er koblet til internett har en unik IP-adresse som andre maskiner bruker for å finne enheten. DNS-servere fjerner behovet for at mennesker må huske IP-adresser, for eksempel 192.168.1.1 (i IPv4), eller mer komplekse nyere alfanumeriske IP-adresser som 2400:cb00:2048:1::c629:d7a2 (i IPv6).
